0015 系统设计基础

Q4 Systems, Projects, Interviews | 预计 45 分钟 | lecture trace

系统设计不是背架构图，而是在约束下解释取舍。

答题框架

clarify requirements
define APIs and flows
model data
estimate scale
draw architecture
choose storage, cache, queue, compute
handle failure, security, observability
explain tradeoffs

常见组件取舍

组件得到代价
缓存低延迟、少打数据库。失效和一致性。
队列削峰、异步、解耦。重复消费和最终一致性。
读副本读扩展。复制延迟。
CDN全球分发。缓存失效复杂度。

练习

  1. 设计一个项目管理系统。
  2. 列出 5 个功能需求和 5 个非功能需求。
  3. 估算 QPS、数据量和峰值。
  4. 说明是否需要缓存、队列、CDN。
交付物：一份系统设计答题稿。

自测

为什么要先澄清需求？

不同需求会导向完全不同设计，先澄清可以避免过度设计或答偏。

什么时候引入队列？

当任务可异步处理、需要削峰、需要解耦或外部服务不稳定时。

系统设计里怎么表现成熟？

主动说明约束、取舍、失败模式和未来演进，而不是堆组件名。
