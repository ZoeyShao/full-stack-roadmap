0010 认证、Session、JWT 与 OAuth

Q2 Backend Fundamentals | 预计 45 分钟 | lecture trace

认证回答“你是谁”，授权回答“你能做什么”。这两个问题混在一起，是很多安全漏洞的起点。

三条线

主题核心问题例子
Authentication你是谁？登录后得到 user id。
Authorization你能做什么？只能删除自己的项目。
Session如何记住登录状态？cookie 里带 session id。

Session、JWT、OAuth

  • Session cookie：服务端保存会话，浏览器带 cookie。
  • JWT：携带 claims 的 token，可签名，但 payload 默认不保密。
  • OAuth：授权框架，让第三方应用在用户同意后访问资源。
login -> verify password -> create session/token
request -> authenticate -> authorize resource action
logout -> revoke/expire session or refresh token

  4. 用 OWASP Top Ten 检查你的 API。

自测

JWT 签名是否等于加密？

不是。签名证明内容未被篡改，但 payload 通常可被读取。

为什么只隐藏前端按钮不算授权？

攻击者可以直接调用 API，真正授权必须在后端检查。

OAuth 主要解决什么？

让用户授权第三方应用访问资源，而不是把密码交给第三方。
