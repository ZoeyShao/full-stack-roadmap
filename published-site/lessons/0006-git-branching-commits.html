0006 Git 分支、提交和协作

Q1 Frontend Fundamentals | 预计 30 分钟 | lecture trace

Git 不是保存按钮。Git 是变更历史数据库。你要学会让别人看懂你为什么改、改了什么、怎么验证。

基本工作流

git switch -c feature/project-form
git status
git add src/ProjectForm.tsx
git commit -m "Add project creation form"
git push -u origin feature/project-form

提交质量

坏提交好提交
updateAdd accessible project form labels
混合格式化、重构和功能每个提交只讲一个变化
没有验证说明PR 里写测试、截图或手动检查

练习

  1. 为 Q1 前端项目创建 feature branch。
  2. 完成一个小功能后提交，不要把无关格式化混进去。
  3. 写一个 PR 描述：背景、变更、验证。
  4. 用 git log --oneline --graph 查看历史。
交付物：一段可 review 的 Git 提交历史。

自测

branch 是什么？

branch 是指向某个 commit 的可移动引用，用来隔离一条开发线。

merge 和 rebase 的差别是什么？

merge 保留分叉历史并产生合并点；rebase 把提交搬到新的 base 上，历史更线性。

为什么提交要小？

小提交更容易 review、回滚、定位 bug，也更能讲清楚开发思路。
