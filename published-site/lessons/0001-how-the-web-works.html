0001 Web 是怎样工作的

Q1 Frontend Fundamentals | 预计 25 分钟 | lecture trace

今天只拿下一件事：当你输入一个 URL，浏览器、网络、服务器和前端代码如何合作，把一串地址变成一个可交互页面。

核心模型

URL -> DNS -> TCP/TLS -> HTTP request -> server
    -> HTTP response -> HTML parser -> DOM
    -> CSS parser -> CSSOM -> render tree
    -> JavaScript -> events -> updates

HTML、CSS、JavaScript 不是同一种东西。HTML 是结构，CSS 是表现，JavaScript 是行为。浏览器把它们组合成 DOM、样式和事件循环。

关键概念

概念你要能解释
URL协议、域名、路径、查询参数分别指向什么。
DNS为什么浏览器需要把域名解析成 IP。
HTTPrequest 和 response 由 method、status、headers、body 组成。
渲染HTML 生成 DOM，CSS 生成 CSSOM，二者合成 render tree。
DevToolsNetwork 面板能看到每个资源的请求、状态码、耗时和响应头。

练习

  1. 打开任意网页，按 F12 打开 DevTools 的 Network。
  2. 刷新页面，找到第一个 document 请求。
  3. 写下它的 URL、method、status code、content-type。
  4. 再找一个 CSS 或 JS 请求，说明它和 document 请求的关系。
交付物：画一张“URL 到页面可交互”的流程图，不超过 10 个节点。

自测

为什么 URL 里有域名还需要 DNS？

因为网络传输最终需要 IP 地址。DNS 把人类容易记的域名解析成机器可路由的地址。

HTTP response 的 status code 说明什么？

说明服务器如何处理请求。比如 200 成功，404 未找到，500 服务器错误。

HTML、CSS、JavaScript 的分工是什么？

HTML 负责结构和语义，CSS 负责呈现和布局，JavaScript 负责交互和动态行为。
