0011 云服务积木：S3、CloudFront、Lambda、API Gateway

Q3 Cloud + DevOps | 预计 40 分钟 | lecture trace

云不是一台别人家的电脑。云是很多托管服务的组合。你要知道每块积木负责什么边界。

典型全栈云架构

Browser
  -> CloudFront
  -> S3 static frontend

Browser
  -> API Gateway
  -> Lambda
  -> Database / external services
  -> CloudWatch logs and metrics

服务定位

服务职责关键风险
S3对象存储，静态资源和上传文件。公开权限和缓存头。
CloudFrontCDN，边缘缓存和分发。缓存失效和 origin 访问。
Lambda事件驱动函数。超时、冷启动、权限。
API GatewayHTTP API 入口。CORS、认证、限流。
IAM身份和权限。权限过宽。

练习

  1. 画出前端静态发布路径。
  2. 画出 API 请求路径。
  3. 为 Lambda 写最小权限说明。
  4. 列出需要记录到 CloudWatch 的日志字段。
交付物：云上全栈架构草图。

自测

S3 bucket 默认是否应该公开？

不应该。默认保持私有，只有明确需要时通过受控方式暴露。

CloudFront 的 origin 是什么？

CloudFront 回源获取内容的源站，如 S3 bucket 或 HTTP server。

Lambda 最大的运维边界是什么？

运行时、超时、内存、冷启动、权限和日志都需要明确设计。
