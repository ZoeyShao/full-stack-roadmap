0007 后端运行时与 HTTP API

Q2 Backend Fundamentals | 预计 35 分钟 | lecture trace

后端服务的最小职责：接收请求，验证输入，执行业务逻辑，访问数据，返回稳定响应，并留下可排查的日志。

请求处理链

request -> route -> middleware -> handler
        -> validation -> business logic
        -> database -> response -> log

后端语言选择

选择适合
Node.js前后端同用 JavaScript/TypeScript，API 和实时应用生态强。
Python脚本、数据处理、AI 生态、快速 API 开发。
GraphQL前端需要灵活声明数据形状，服务端能维护 schema。

练习

  1. 实现 GET /health 返回 { "ok": true }。
  2. 实现 GET /projects 返回项目数组。
  3. 统一错误响应格式：{ "error": { "message": "..." } }。
  4. 每次请求记录 method、path、status、duration。
交付物：一个最小 HTTP API 服务。

自测

route handler 负责什么？

负责处理匹配到的请求，把输入转成业务动作，并返回响应。

middleware 适合放什么？

适合放日志、认证、解析 body、错误处理等跨路由通用逻辑。

为什么错误响应要统一？

前端和调用方可以稳定处理错误，日志和排障也更一致。
