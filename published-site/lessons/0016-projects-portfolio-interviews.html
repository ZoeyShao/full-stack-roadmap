0016 项目、作品集和面试表达

Q4 Systems, Projects, Interviews | 预计 40 分钟 | lecture trace

项目不是写完就结束。你要把它整理成别人能运行、能理解、能信任、能追问的证据。

项目展示包

材料必须说明
README问题、功能、技术栈、运行方式、截图、部署地址。
Demo2 分钟核心用户流程和技术亮点。
架构图前端、API、数据库、云服务、CI/CD、监控。
作品集页你的职责、技术取舍、结果和链接。

简历 bullet

使用 React、Node.js 和 PostgreSQL 构建项目管理系统，
实现认证、CRUD、自动部署和基础监控，
将关键用户流程端到端响应控制在 300ms 内。

好的 bullet 写“做了什么、用了什么、结果如何、约束是什么”。不要只写技术名词清单。

练习

  1. 给一个项目补 README 大纲。
  2. 写 2 分钟 Demo 脚本。
  3. 为项目写 3 条简历 bullet。
  4. 准备 5 个面试追问：难点、取舍、失败、测试、改进。
交付物：作品集项目页、Demo 脚本和简历 bullet。

自测

为什么作品集不能只放截图？

截图不能证明你能设计、实现、部署和维护系统，需要代码、说明、Demo 和取舍。

项目面试最怕什么？

只会背技术栈，讲不清为什么这样设计、失败过什么、如何验证质量。

Demo 最重要的顺序是什么？

先讲用户问题，再演示核心流程，最后讲技术取舍和结果。
