AWS 积木

服务一句话项目中怎么用常见坑
S3对象存储存静态文件、上传文件、备份。公开权限、bucket policy、缓存头。
CloudFrontCDN 分发前端静态资源、图片、API 缓存。缓存失效、origin 权限、HTTPS 证书。
Lambda事件驱动函数小型 API、队列消费者、定时任务。冷启动、超时、权限、日志。
API GatewayAPI 入口HTTP 路由到 Lambda 或后端。CORS、认证、限流、错误映射。
CloudWatch观测平台日志、指标、告警、仪表盘。日志成本、告警噪音、缺少结构化字段。
IAM身份和权限控制服务和用户能访问什么。权限过宽、长期密钥、角色信任关系。

Terraform 心智模型

write .tf files
  -> terraform init
  -> terraform plan
  -> review diff
  -> terraform apply
  -> state records real resources
对象含义
providerTerraform 和云平台 API 的适配器，如 AWS provider。
resource要创建和管理的真实资源，如 S3 bucket。
data source读取已有资源或外部信息，不直接创建。
state代码资源和真实资源的映射，必须保护。
module可复用基础设施组件。

CI/CD 最小流水线

on pull request:
  install
  lint
  typecheck
  test
  build

on merge to main:
  install
  test
  build
  deploy staging
  smoke test

流水线的目的不是仪式感，而是把“我机器上能跑”变成“每次改动都可验证”。

观测性三件套

信号回答例子

Metrics整体趋势如何？QPS、p95 latency、5xx rate、queue depth。
