0004 React + TypeScript 组件模型

Q1 Frontend Fundamentals | 预计 40 分钟 | lecture trace

React 的核心不是 JSX 语法，而是把 UI 拆成有输入、状态和输出的组件树。TypeScript 让这些边界更明确。

组件接口

type ProjectCardProps = {
  title: string;
  status: "draft" | "active" | "archived";
  onArchive: () => void;
};

function ProjectCard({ title, status, onArchive }: ProjectCardProps) {
  return (
    <article>
      <h2>{title}</h2>
      <p>{status}</p>
      <button onClick={onArchive}>Archive</button>
    </article>
  );
}

props 是组件的公共接口。state 是组件记住的内部变化。事件 handler 是用户动作进入系统的入口。

拆组件步骤

  1. 圈出页面中的视觉区域。
  2. 给每个区域命名，名字来自业务，而不是颜色和位置。
  3. 判断哪些数据通过 props 传入。
  4. 判断哪些状态会随用户操作变化。
  5. 把可复用且责任清楚的区域提成组件。

练习

  1. 把项目列表页面拆成 ProjectPage、ProjectFilter、ProjectList、ProjectCard。
  2. 为项目状态写联合类型。
  3. 给归档按钮传入 callback prop。
  4. 解释状态为什么放在当前组件，而不是更深或更浅。
交付物：一个有类型的 React 组件树。

自测

props 和 state 最大区别是什么？

props 来自父组件，是只读输入；state 属于当前组件，会随交互变化并触发渲染。

为什么列表渲染需要 key？

key 帮 React 识别列表项在插入、删除、重排时的身份。

联合类型适合表达什么？

适合表达几个互斥的状态或变体，例如项目状态 draft/active/archived。
