0012 Infrastructure as Code 与 Terraform

Q3 Cloud + DevOps | 预计 40 分钟 | lecture trace

IaC 的价值是让基础设施可以 review、复现和回滚，而不是在控制台里靠记忆点点点。

Terraform 流程

terraform init
terraform fmt
terraform validate
terraform plan
terraform apply

核心对象

对象含义
provider连接云平台 API。
resource声明要管理的真实资源。
data source读取已有资源或外部信息。
state记录代码和真实资源映射。
module可复用基础设施组件。

练习

  1. 写一个 S3 bucket 的 Terraform 草稿。
  2. 说明 plan 输出应该如何 review。
  3. 解释 state 为什么不能随便丢或公开。
  4. 把 dev 和 prod 的变量分开。
交付物：一份 Terraform 计划说明。

自测

Terraform state 记录什么？

记录 Terraform 配置资源和真实云资源之间的映射及属性。

为什么 apply 前要看 plan？

plan 是变更预览，可以避免误删、误改和权限扩大。

module 什么时候值得抽？

当一组资源有稳定边界并会被多处复用时。
