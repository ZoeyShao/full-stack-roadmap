0002 HTML、CSS、JavaScript 基础闭环

Q1 Frontend Fundamentals | 预计 35 分钟 | lecture trace

这节课把三件事连成闭环：HTML 放内容，CSS 让内容可读，JavaScript 让用户动作产生反馈。

三层分工

层职责失败表现
HTML表达页面结构和语义。全是 div，表单没有 label，标题层级混乱。
CSS控制布局、间距、颜色、字体、状态。样式互相覆盖，移动端溢出，按钮没有焦点状态。
JavaScript响应事件、读写状态、更新 DOM、发请求。点击没反馈，表单刷新丢数据，错误不可见。

最小交互

<form data-form>
  <label>Name <input name="name"></label>
  <button>Save</button>
</form>
<p data-output></p>

<script>
const form = document.querySelector("[data-form]");
const output = document.querySelector("[data-output]");

form.addEventListener("submit", (event) => {
  event.preventDefault();
  const data = new FormData(form);
  output.textContent = `Saved: ${data.get("name")}`;
});
</script>

你要能解释每一行：选择 DOM、监听事件、阻止默认提交、读取表单、更新页面。

练习

  1. 写一个个人项目登记表：项目名、技术栈、项目状态。
  2. 用 CSS 让表单在 360px 宽度下不溢出。
  3. 提交后在页面下方显示一张项目卡片。
  4. 空项目名时显示文字错误，不只改变颜色。
交付物：一个无框架 HTML/CSS/JS 页面。

自测

为什么按钮动作应该用 button 而不是 div？

button 自带可点击、可聚焦、键盘触发和语义。div 需要额外补大量可访问性行为。

event.preventDefault() 在表单里常用来做什么？

阻止浏览器默认提交刷新页面，让 JavaScript 接管验证、请求和反馈。

CSS cascade 是什么？

浏览器根据来源、选择器优先级、顺序等规则决定最终应用哪条样式。
