Q1 Frontend Fundamentals | 预计 40 分钟 | lecture trace

真实前端不是只渲染静态组件，而是处理输入、请求、错误、缓存和用户可见的反馈。

状态分类

状态例子工具
本地 UI 状态弹窗是否打开、输入框内容。useState、useReducer
派生状态过滤后的列表、总价。渲染时计算，不单独存。
服务端状态项目列表、当前用户。TanStack Query 等缓存工具。
URL 状态搜索词、分页页码。query string 或 router。

TanStack Query 模型

const projectsQuery = useQuery({
  queryKey: ["projects", filter],
  queryFn: () => fetchProjects(filter),
});

const createProject = useMutation({
  mutationFn: createProjectRequest,
  onSuccess: () => queryClient.invalidateQueries({ queryKey: ["projects"] }),
});

queryKey 是缓存身份。改变服务器数据后，需要让相关缓存失效，否则 UI 可能显示旧数据。

自测

为什么不要把 filteredItems 存进 state？

它可以从原始列表和筛选条件计算出来，重复存储会带来同步错误。

mutation 成功后为什么常常要 invalidate query？

因为服务器数据已经改变，相关缓存需要重新获取或更新。

好前端测试更关注什么？

关注用户能看到和能完成的行为，而不是组件内部实现细节。
