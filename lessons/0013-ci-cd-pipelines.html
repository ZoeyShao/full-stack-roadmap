0013 CI/CD 流水线

Q3 Cloud + DevOps | 预计 35 分钟 | lecture trace

CI/CD 把“我觉得没问题”变成“每次变更都有自动证据”。

最小流水线

pull_request:
  install
  lint
  typecheck
  test
  build

main:
  install
  test
  build
  deploy
  smoke-test

GitHub Actions 词汇

词含义
workflow一条自动化流程。
job在 runner 上运行的一组 step。
step一个命令或 action。
secret敏感配置，如 token 和云凭据。
environment部署目标，如 staging、production。

练习

  1. 写 PR 检查：lint、typecheck、test、build。
  2. 写 main 部署：build 后发布到 staging。
  3. 列出哪些值必须放 secret。
  4. 设计失败后如何回滚。
交付物：一份 GitHub Actions workflow 草稿。

自测

CI 和 CD 的区别是什么？

CI 关注集成验证，CD 关注把验证过的变更交付或部署到环境。

为什么不要在 workflow 里明文写密钥？

仓库历史、日志和 fork 都可能泄露密钥，应该用 secret 或 OIDC。

smoke test 是什么？

部署后跑的最小关键路径检查，用来确认服务基本可用。
