0003 布局、响应式和可访问性

Q1 Frontend Fundamentals | 预计 35 分钟 | lecture trace

你今天要学会一个判断：Flex 解决一维排列，Grid 解决二维布局，响应式和可访问性不是最后补丁，而是布局设计的一部分。

布局选择

场景工具原因
导航栏、按钮组、表单行Flexbox沿一个方向排列，对齐和分布。
页面主栏 + 侧栏、卡片矩阵Grid同时控制行和列。
不同屏幕尺寸Media query先小屏可用，再增强大屏。
 
.cards {
  display: grid;
  grid-template-columns: repeat(auto-fit, minmax(220px, 1fr));
  gap: 16px;
}

.actions {
  display: flex;
  gap: 12px;
  flex-wrap: wrap;
}

可访问性底线

  • 所有可操作元素都能通过键盘到达。
  • 输入框有 label，错误信息有文字。
  • 焦点状态清楚，不要移除 outline 后不给替代。
  • 图片 alt 写信息，不写装饰噪音。
  • 颜色不是唯一提示。

练习

  1. 做一个项目卡片列表，小屏一列，大屏多列。
  2. 顶部工具条包含搜索框和两个按钮，窄屏自动换行。
  3. 用 Tab 从搜索框走到每个按钮，确认焦点可见。
  4. 给空搜索提交显示文字错误。
交付物：一个响应式且键盘可用的页面。

自测

什么时候不用 Grid？

当你只需要沿一条轴排列和对齐时，Flexbox 通常更直接。

mobile-first 是什么？

默认样式先服务小屏，再用 media query 为更大屏幕增加布局能力。

为什么错误提示不能只用红色？

色弱、读屏或高对比模式用户可能无法获得信息，需要文字说明和语义关联。
