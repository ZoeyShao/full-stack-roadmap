HTML 先问语义

需求优先元素原因
页面主标题<h1>表达文档结构，不是只把字变大。
导航<nav> + <a>可被浏览器、搜索引擎和辅助技术识别。
主要内容<main>每页通常一个主内容区。
表单输入<label> + <input>点击 label 能聚焦输入框，读屏能读出字段含义。
按钮动作<button>天然支持键盘、焦点和点击语义。

CSS 布局决策

问题首选工具判断规则
一行按钮如何分布Flexbox沿一个轴对齐、分布、换行。
页面整体左右栏Grid同时关心行和列，使用明确轨道。
内容随容器变宽max-width + width先限制阅读宽度，再居中。
移动端布局Mobile-first media query默认小屏，再逐步增强大屏。
 
.page {
  width: min(100% - 32px, 960px);
  margin-inline: auto;
}

.cards {
  display: grid;
  grid-template-columns: repeat(auto-fit, minmax(220px, 1fr));
  gap: 16px;
}

.toolbar {
  display: flex;
  align-items: center;
  justify-content: space-between;
  gap: 12px;
  flex-wrap: wrap;
}

JavaScript 最小交互

const form = document.querySelector("[data-form]");
const output = document.querySelector("[data-output]");

form.addEventListener("submit", (event) => {
  event.preventDefault();
  const data = new FormData(form);
  output.textContent = `你好，${data.get("name")}`;
});

能解释这段代码，就已经连通了 DOM 查询、事件监听、默认行为、表单数据和页面更新。

可访问性底线

检查项合格标准
键盘Tab 能到达所有可操作控件，Enter/Space 能触发按钮。
标签每个输入都有可见 label 或清晰的 accessible name。
错误错误信息要靠文字说明，不只靠红色。
图片信息图片写 alt，装饰图片空 alt。
对比度正文和背景有足够对比，焦点状态清楚。
